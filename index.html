X AI 回复助手隐私政策

本政策说明 X AI 回复助手 Chrome 扩展在生成回复草稿时如何处理数据。

生效日期：2026 年 7 月 13 日最后更新：2026 年 7 月 21 日

一句话说明：用户选择并授权云端或本机 AI 接口、明确同意数据处理后，进入 X 单条帖子页面会自动读取该帖上下文，并调用所选接口生成可引导、可复制的回复草稿。扩展不会自动发布回复，不运行开发者中转服务器，也不出售用户数据。

1. 扩展用途与处理时机

X AI 回复助手的单一用途，是根据当前 X 单条帖子及相关评论生成多种风格的回复草稿。扩展不会在后台持续收集浏览历史；在用户完成设置后，进入单条帖子页面会自动为这一次生成操作读取必要上下文。用户在右侧点击重新生成时会发起新的请求。

2. 处理的数据

一次生成操作可能处理以下信息：

  • 用户主动选择分析的当前 X 帖子文本；
  • 用户正在回复的评论，以及页面上可见的相关评论；
  • 当前页面 URL 与页面中公开显示的作者名称或用户名；
  • 用户在右侧输入的本次 AI 引导内容；
  • 用户自行填写的 AI 服务、接口地址、模型名称、认证方式和 API Key（如接口需要）。

扩展不会读取与当前回复任务无关的账号密码、私信或剪贴板内容。

3. 数据用途与用户选择的 AI 接口

页面上下文和本次 AI 引导仅用于生成用户请求的回复草稿，并直接发送到用户在扩展弹窗中选择和授权的接口。该接口可以是 DeepSeek、OpenAI、Anthropic、Gemini、OpenRouter、其他兼容云服务、本机 Ollama、LM Studio，或用户自行配置的本机、局域网和自建服务。API Key 仅作为所选接口的认证信息使用。

云端接口由对应服务商处理数据并可能消耗用户自己的 API 额度；本机或局域网接口由该地址对应的软件或设备处理。远程服务建议使用 HTTPS；HTTP 主要用于用户明确配置的本机或局域网服务。对应接口对请求数据的处理受其自身政策和用户配置约束。

4. 本地存储

扩展仅在 Chrome 扩展本地存储中保存所选 AI 服务、接口地址、模型、认证方式、API Key 和用户的数据处理同意状态，不使用 Chrome Sync。这些数据不会同步到开发者服务器。扩展开发者不运行中转服务器，也不接收或保存用户的 API Key、帖子内容、评论内容、引导内容或生成结果。

5. 接口权限、剪贴板与发布

为支持自定义云端或本机 AI，扩展声明可选的 HTTP/HTTPS 主机权限。安装时不会因此获得所有网站访问权；只有用户点击“保存并授权此接口”后，Chrome 才会针对所填写的接口主机请求许可。该权限只用于从扩展后台向已保存的 AI 接口发起生成请求，不用于读取其他网站页面。

只有在用户点击“复制文案”后，所选草稿才会写入剪贴板。扩展不会读取剪贴板内容，不会自动填写 X 评论框，也不会代替用户发布、点赞或执行其他账号操作。

6. 数据保留与删除

扩展自身不保留帖子上下文、本次 AI 引导或生成结果。用户可以在扩展弹窗中点击“清除本地设置”，删除本地保存的 AI 配置、API Key 和同意状态；卸载扩展也会清除扩展的本地数据。第三方或自建接口对 API 请求数据的保留由对应政策和用户配置决定。

7. 数据共享与有限使用

除为完成回复生成而直接传输至用户选择的 AI 接口外，扩展不会向其他第三方出售、出租或共享用户数据。用户数据不会用于广告、再营销、用户画像、信用评估或与扩展单一用途无关的目的，也不会允许开发者人工读取用户数据。

本扩展对用户数据的使用遵守 Chrome Web Store User Data Policy 的 Limited Use 要求，仅限于提供或改进公开说明的单一功能。

8. 用户选择

用户可以选择云端或本机接口、编辑接口地址和模型、撤销接口主机权限、不勾选自动数据处理同意、在扩展弹窗中清除本地设置或卸载扩展。配置未完成或未同意新版数据说明时，扩展不会调用 AI 接口生成草稿。

9. 政策更新与联系

如果扩展的数据处理方式发生实质变化，本页面将更新相关说明及“最后更新”日期。如对本政策或扩展的数据处理方式有疑问，请通过 X AI 回复助手 Chrome Web Store 商品页公布的开发者支持渠道，或在 公开支持页面联系开发者。
