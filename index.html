X AI 回复助手隐私政策

本政策说明 X AI 回复助手 Chrome 扩展在生成回复草稿时如何处理数据。

生效日期：2026 年 7 月 13 日最后更新：2026 年 7 月 17 日

一句话说明：用户保存 API Key 并明确同意数据处理后，进入 X 单条帖子页面会自动读取该帖上下文，并调用用户自己的 DeepSeek API 生成可引导、可复制的回复草稿。扩展不会自动发布回复，不运行开发者中转服务器，也不出售用户数据。

1. 扩展用途与处理时机

X AI 回复助手的单一用途，是根据当前 X 单条帖子及相关评论生成多种风格的回复草稿。扩展不会在后台持续收集浏览历史；在用户完成设置后，进入单条帖子页面会自动为这一次生成操作读取必要上下文。用户在右侧点击重新生成时会发起新的请求。

2. 处理的数据

一次生成操作可能处理以下信息：

  • 用户主动选择分析的当前 X 帖子文本；
  • 用户正在回复的评论，以及页面上可见的相关评论；
  • 当前页面 URL 与页面中公开显示的作者名称或用户名；
  • 用户在右侧输入的本次 AI 引导内容；
  • 用户自行提供的 DeepSeek API Key。

扩展不会读取与当前回复任务无关的账号密码、私信或剪贴板内容。

3. 数据用途与 DeepSeek

页面上下文和本次 AI 引导仅用于生成用户请求的回复草稿。扩展通过 HTTPS 将必要上下文和 API 请求直接发送至 DeepSeek API；DeepSeek 是完成该功能所必需的第三方服务提供商。API Key 仅作为调用 DeepSeek API 的身份验证信息使用。每次自动生成或重新生成都会消耗用户自己的 DeepSeek API 额度。

DeepSeek 对收到数据的处理受其自身政策约束。请阅读 DeepSeek 隐私政策。

4. 本地存储

扩展仅在 Chrome 扩展本地存储中保存 DeepSeek API Key 和用户的数据处理同意状态。这些数据不会同步到开发者服务器。扩展开发者不运行中转服务器，也不接收或保存用户的 API Key、帖子内容、评论内容或生成结果。

5. 剪贴板与发布

只有在用户点击“复制文案”后，所选草稿才会写入剪贴板。扩展不会读取剪贴板内容，不会自动填写 X 评论框，也不会代替用户发布、点赞或执行其他账号操作。

6. 数据保留与删除

扩展自身不保留帖子上下文、本次 AI 引导或生成结果。用户可以在扩展弹窗中点击“清除”，删除本地保存的 API Key 和同意状态；卸载扩展也会清除扩展的本地数据。DeepSeek 对 API 请求数据的保留由其隐私政策和用户的 DeepSeek 服务设置决定。

7. 数据共享与有限使用

除为完成回复生成而直接传输至 DeepSeek 外，扩展不会向其他第三方出售、出租或共享用户数据。用户数据不会用于广告、再营销、用户画像、信用评估或与扩展单一用途无关的目的，也不会允许人工读取用户数据。

本扩展对用户数据的使用遵守 Chrome Web Store User Data Policy 的 Limited Use 要求，仅限于提供或改进公开说明的单一功能。

8. 用户选择

用户可以不提供 DeepSeek API Key、不勾选自动数据处理同意、在扩展弹窗中清除本地设置或卸载扩展。未提供 API Key 或未同意新版数据说明时，扩展不会调用 DeepSeek 生成草稿。

9. 政策更新与联系

如果扩展的数据处理方式发生实质变化，本页面将更新相关说明及“最后更新”日期。如对本政策或扩展的数据处理方式有疑问，请通过 X AI 回复助手 Chrome Web Store 商品页公布的开发者支持渠道，或在 公开支持页面联系开发者。
